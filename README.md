# Amazon SDE Interview Prep

A practical preparation guide for candidates interviewing for Amazon Software Development Engineer roles, including Amazon Leo / satellite-infrastructure-related SDE roles.

This repository is designed for candidates who are preparing for:
- Amazon SDE interviews
- 4-round virtual interview loops
- Leadership Principles behavioral interviews
- DSA / coding rounds
- Project deep-dive discussions
- Recruiter conversations around role status, team matching, and work authorization

> Disclaimer: This is independent educational material. It is not official Amazon guidance and is not legal or immigration advice.

---

## Why This Repository Exists

Many candidates face confusion during the Amazon process:

- The Amazon portal says “No longer under consideration”
- A recruiter still sends an interview invite
- The interview email says “Software Development Engineer” but not the original team name
- The loop has 4 rounds of 60 minutes each
- The candidate is unsure how much to prepare for coding vs behavioral vs project discussion
- International students on F-1/OPT are unsure how to discuss work authorization professionally

This repo helps you prepare in a structured way.

## What You Should Prepare

### Coding

Focus on patterns, not random memorization:

- Arrays and hash maps
- Sliding window
- Two pointers
- Binary search
- Trees
- Graphs
- BFS / DFS
- Heaps
- Intervals
- Dynamic programming basics
- Recursion and backtracking

### Behavioral

Prepare 8–10 strong stories using the STAR format:

- Situation
- Task
- Action
- Result

Each story should map to one or more Amazon Leadership Principles.

### Project Deep Dive

Be ready to explain:

- Architecture
- Trade-offs
- Scalability
- Failure handling
- Testing
- Monitoring
- Security
- What you would improve today

- Day 12: Behavioral mock
- Day 13: Full coding mock
- Day 14: Final review and recruiter questions

## Best Mindset

Amazon interviews usually test more than code.

They test whether you can:

- Clarify ambiguity
- Communicate clearly
- Handle edge cases
- Think about customers
- Take ownership
- Dive deep into technical problems
- Make practical trade-offs
- Deliver reliable systems

Prepare like an engineer, not like someone memorizing questions.
