
# DSA Pattern Roadmap for Amazon SDE Interviews

The goal is not to solve 500 random problems.

The goal is to recognize patterns quickly.

---

## Pattern 1: Arrays and Hash Maps

### When to Use

Use hash maps when you need:

- Fast lookup
- Frequency counting
- Duplicate detection
- Pair finding
- Grouping

### Common Problems

- Two sum
- Group anagrams
- Longest consecutive sequence
- Subarray sum
- Frequency-based sorting

### Recognition Clues

- “Find pairs”
- “Count occurrences”
- “Check duplicates”
- “Group similar items”
- “Find first unique”

---

## Pattern 2: Sliding Window

### When to Use

Use sliding window when working with:

- Contiguous subarrays
- Substrings
- Maximum or minimum window
- Fixed-size window
- Variable-size window

### Common Problems

- Longest substring without repeating characters
- Minimum window substring
- Maximum sum subarray of size K
- Longest repeating character replacement

### Recognition Clues

- “Longest substring”
- “Smallest window”
- “Contiguous”
- “At most K”
- “Subarray of size K”

---

## Pattern 3: Two Pointers

### When to Use

Use two pointers when:

- Array is sorted
- You need pair comparison
- You need in-place operations
- You need to shrink from both ends

### Common Problems

- Valid palindrome
- Container with most water
- 3Sum
- Remove duplicates
- Merge sorted arrays

### Recognition Clues

- “Sorted array”
- “Pair”
- “In-place”
- “Left and right”
- “Palindrome”

---

## Pattern 4: Binary Search

### When to Use

Use binary search when:

- Search space is sorted
- You can eliminate half
- You need minimum possible answer
- You need maximum feasible answer

### Common Problems

- Search in rotated sorted array
- Find minimum in rotated array
- First and last position
- Search insert position
- Capacity to ship packages

### Recognition Clues

- “Sorted”
- “Minimum possible”
- “Maximum possible”
- “Find boundary”
- “Can we do this within X?”

---

## Pattern 5: Trees, DFS, and BFS

### DFS

Use DFS for:

- Path problems
- Recursion
- Tree depth
- Subtree checks
- Backtracking-style traversal

### BFS

Use BFS for:

- Level order traversal
- Shortest path in unweighted structures
- Nearest target
- Minimum steps

### Common Problems

- Maximum depth of binary tree
- Validate binary search tree
- Lowest common ancestor
- Binary tree level order traversal
- Serialize and deserialize binary tree

---

## Pattern 6: Graphs

### When to Use

Use graph techniques when the problem has:

- Nodes and edges
- Dependencies
- Connectivity
- Paths
- Cycles
- Components

### Common Problems

- Number of islands
- Course schedule
- Clone graph
- Pacific Atlantic water flow
- Shortest path in grid

### Recognition Clues

- “Connected”
- “Can reach”
- “Dependencies”
- “Network”
- “Grid”
- “Cycle”

---

## Pattern 7: Heaps and Priority Queues

### When to Use

Use heaps when you need:

- Top K elements
- Running median
- Scheduling
- Repeatedly getting min or max

### Common Problems

- Kth largest element
- Top K frequent elements
- Merge K sorted lists
- Meeting rooms
- Task scheduler

### Recognition Clues

- “Top K”
- “Smallest K”
- “Largest K”
- “Most frequent”
- “Merge sorted”

---

## Pattern 8: Intervals

### When to Use

Use intervals when problems involve:

- Time ranges
- Bookings
- Merging overlaps
- Scheduling
- Conflicts

### Common Problems

- Merge intervals
- Insert interval
- Meeting rooms
- Non-overlapping intervals

### Recognition Clues

- “Start and end”
- “Overlapping”
- “Schedule”
- “Calendar”
- “Meeting”

---

## Pattern 9: Dynamic Programming

### When to Use

Use DP when:

- Problem has overlapping subproblems
- You need optimal result
- Choice at one step affects future
- Brute force repeats work

### Common Problems

- Climbing stairs
- House robber
- Coin change
- Longest increasing subsequence
- Longest common subsequence

### Recognition Clues

- “Maximum ways”
- “Minimum cost”
- “Can you form”
- “Optimal”
- “Choose or skip”

## How to Practice

For each problem:

1. Identify the pattern.
2. Explain brute force.
3. Optimize.
4. Write clean code.
5. Test edge cases.
6. Explain time and space complexity.

---

## Best Rule

If you cannot explain the pattern, you have not learned the problem yet.
