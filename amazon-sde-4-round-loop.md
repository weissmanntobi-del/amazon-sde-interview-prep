# Amazon SDE 4-Round Interview Loop

Amazon SDE interview loops can vary by level, team, location, and hiring pipeline.

A common virtual loop may include 4 rounds of around 60 minutes each.

---

## Typical Round Structure

### Round 1: Coding Interview

Expected focus:

- Problem solving
- Data structures
- Algorithms
- Edge cases
- Clean code
- Communication

You may be asked to solve one or two coding problems.

Common topics:

- Arrays
- Strings
- Hash maps
- Trees
- Graphs
- Recursion
- Binary search
- Sliding window
- Heaps

---

### Round 2: Coding + Leadership Principles

This round may combine coding and behavioral questions.

Example structure:

- 5–10 minutes: introduction
- 35–40 minutes: coding
- 10–15 minutes: behavioral question

Prepare to explain:

- Your approach
- Time complexity
- Space complexity
- Edge cases
- Alternative solutions

---

### Round 3: Technical Deep Dive / Project Discussion

This round may focus on your past projects.

You should be ready to explain:

- What you built
- Why you built it
- Architecture
- Design decisions
- Trade-offs
- Failure cases
- Testing
- Deployment
- Monitoring
- What you would improve

For experienced candidates, this may include system design.

For new grads or junior candidates, this may be more project-focused.

---

### Round 4: Behavioral / Bar Raiser Style Round

This round may heavily focus on Amazon Leadership Principles.

Expect questions like:

- Tell me about a time you failed.
- Tell me about a time you disagreed with someone.
- Tell me about a time you had to deliver under pressure.
- Tell me about a time you solved an ambiguous problem.
- Tell me about a time you improved a system.
- Tell me about a time you made a decision with incomplete information.

Use the STAR format.

---

## What Interviewers Usually Look For

### Coding Signal

- Can you understand the problem?
- Can you ask clarifying questions?
- Can you identify the right data structure?
- Can you handle edge cases?
- Can you write clean code?
- Can you explain complexity?

### Behavioral Signal

- Do you take ownership?
- Do you learn from mistakes?
- Do you handle ambiguity?
- Do you communicate clearly?
- Do you care about customers?
- Do you raise the quality bar?

### Engineering Signal

- Can you build reliable systems?
- Can you debug deeply?
- Can you make trade-offs?
- Can you operate software in production?
- Can you simplify complexity?

---

## How to Prepare for Each Round

| Round | Main Focus | Preparation |
|---|---|---|
| Coding 1 | DSA fundamentals | Arrays, strings, hash maps, trees |
| Coding 2 | DSA + communication | Timed mock interviews |
| Technical Deep Dive | Projects / design | Prepare 2 strong projects |
| Behavioral | Leadership Principles | Prepare 8–10 STAR stories |

---

## 60-Minute Coding Round Strategy

### First 5 Minutes

- Read the problem carefully
- Ask clarifying questions
- Confirm inputs and outputs
- Discuss edge cases

### Next 10 Minutes

- Explain brute force
- Improve approach
- Choose data structure
- Walk through example

### Next 30 Minutes

- Write clean code
- Think aloud
- Avoid silent coding
- Test as you go

### Final 10–15 Minutes

- Test edge cases
- Explain complexity
- Discuss improvements
- Handle follow-up questions

---

## Common Mistakes

Avoid:

- Jumping into code too fast
- Ignoring edge cases
- Staying silent
- Memorizing without understanding
- Not testing your code
- Giving vague behavioral answers
- Blaming teammates in stories
- Not knowing your own projects deeply

---

## Best Preparation Advice

Do not prepare only for LeetCode.

Prepare for the full signal:

- Coding
- Communication
- Ownership
- Debugging
- Project depth
- Leadership Principles
